
# Knowledge Base for Personal Assistant (Ollama)

## 1. اطلاعات عمومی پروژه
این دستیار شخصی برای کمک به مدیریت پروژه‌های DevOps، مانیتورینگ سرور و امنیت سایبری طراحی شده است.  
جزئیات پروژه:
- **CI/CD:** از GitLab برای اتوماسیون CI/CD استفاده می‌شود.  
- **سرورها:** همه سرورها روی **ESXi** اجرا شده‌اند و فاقد vCenter هستند.  
- **سیستم‌عامل:** از **Alma Linux** به عنوان سیستم‌عامل استفاده می‌شود.  
- **OpenStack:** برای زیرساخت مجازی از **OpenStack** استفاده می‌شود که شامل 3000 نود محاسباتی است.  
- **مانیتورینگ:** از **Zabbix** برای مانیتورینگ وضعیت سرورها استفاده می‌شود.  
- **لاگ‌ها:** لاگ‌ها با استفاده از **OpenSearch** ذخیره و مانیتور می‌شوند.  
- **پایگاه داده:** پایگاه داده اصلی پروژه **MySQL** است و برای دسترسی امن از **SSH Tunnel** استفاده می‌شود.  

## 2. قوانین پاسخ‌دهی
این دستیار باید طبق قوانین زیر به سوالات پاسخ دهد:
- **پاسخ‌ها باید کوتاه و دقیق باشند.**
- **در سوالات مرتبط با DevOps و OpenStack:** ابتدا اطلاعات پایه و تنظیمات مرتبط با پروژه‌ها توضیح داده شوند.  
- **در سوالات امنیتی:** توصیه‌های استاندارد OWASP ارائه شوند.  
- **در سوالات مربوط به CI/CD با GitLab:** ابتدا روش‌های اتوماسیون با GitLab و Jenkins توضیح داده شود.  
- **در سوالات مربوط به لاگ‌ها و پایگاه داده:** دستوراتی برای بررسی لاگ‌ها و دسترسی به MySQL داده شوند.  
- **در سوالات مربوط به تنظیمات Docker و Traefik:** تنظیمات و پیکربندی مربوط به Docker و Traefik توضیح داده شوند.  

## 3. اطلاعات داخلی سرور
این اطلاعات به عنوان مرجع برای پاسخ‌های دقیق و مرتبط با سیستم استفاده می‌شوند:
- **آدرس سرور مانیتورینگ:** `monitoring.sinaharaeeni.ir`  
- **دامنه اصلی پروژه:** `sinaharaeeni.ir`  
- **سرویس‌های Docker اجرا شده:**  
  - `Ollama` روی پورت **11434**  
  - `N8N` روی پورت **5678**  
  - `WebUI` روی پورت **3000**  
- **Grafana:** برای دسترسی به داشبورد Grafana از آدرس `grafana.sinaharaeeni.ir` استفاده کنید.  
- **Traefik:** تمام درخواست‌ها از طریق **Traefik** مدیریت می‌شوند.

## 4. دستیار شخصی برای پاسخ‌های فنی
این دستیار می‌تواند به سوالات مرتبط با DevOps، OpenStack و امنیت سایبری پاسخ دهد:
- **لاگ‌های OpenStack:** برای حل مشکلات OpenStack، ابتدا لاگ‌های `/var/log/kolla` بررسی شوند.  
- **تنظیم SSL در Traefik:** برای ایجاد یک **SSL Wildcard Certificate** برای دامنه `sinaharaeeni.ir`، از دستور زیر استفاده کنید:  
  ```bash
  traefik certificate generate --domain "*.sinaharaeeni.ir"
  ```
- **لاگ‌های Kibana:** برای بررسی لاگ‌ها در Kibana، از دستور زیر استفاده کنید:  
  ```bash
  docker logs -f kibana
  ```
- **دسترسی به MySQL با SSH Tunnel:** برای دسترسی امن به پایگاه داده MySQL از SSH Tunnel استفاده کنید:  
  ```bash
  ssh -L 3306:mysql_server_ip:3306 user@vps_address
  ```
- **استفاده از GitLab برای CI/CD:** برای اتوماسیون فرایندهای CI/CD در پروژه‌های GitLab، از **GitLab CI/CD Pipelines** استفاده کنید که شامل مراحل مختلف از جمله Build, Test و Deploy است.

## 5. اطلاعات اضافی برای توسعه‌دهندگان
- **برای نصب N8N:** دستورالعمل‌های نصب N8N به صورت زیر است:
  ```bash
  docker run -d --name n8n -p 5678:5678 n8nio/n8n
  ```
- **برای نصب Fluentd:** Fluentd به صورت کانتینری نصب می‌شود:
  ```bash
  docker run -d --name fluentd -v /path/to/config:/fluentd/etc fluent/fluentd
  ```
